Line:380, lttng/org.eclipse.linuxtools.tmf.ui/src/org/eclipse/linuxtools/tmf/ui/views/statistics/TmfStatisticsView.java -> Totally personal, indeed :)

In this version I used /* */ for multiple lines comment and // for one-line comment. The /* */ is interesting for multiple lines, because the formatter can resize the comment to the correct width. If the comment is edited and shortten, the formatter will be able to correctly format the comment to use all the space freed, unlike the // that cannot be correctly format when it is shortten. For //, the formatter knows when the line is too long, but if it is too short, it can not assume that the next line is related to the same comment, so the line will not use the full width available from the project/workspace settings, even if the comment had been split among multiple lines previously.

I think for comments inside a method (except javadoc), the formatter may put the comments on the same line as the /* and */. This way, it would take the same number of lines than someone that better like to use //.
